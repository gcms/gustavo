Automated Setup of Multi-Cloud Environments for Microservices-Based Applications

 

Gustavo Sousa - Walter Rudametkin - Laurence Duchien

Spirals - Université Lille 1 / CRIStAL / École Centrale de Lille / Inria

 

Summary

 

This accompanying site contains additional information about the research conducted on the use of software product lines tools and ontology reasoning for automated setup of multi-cloud environments for microservices-based applications. Here we make available the developed modeling tools, code base of the implementation and experimentation, as well as the resulting data obtained. All the files and paths mentioned here can be found in the bundle zip.

 

Modeling tools

 

The domain specific languages and editor was developed using Xtext and EMF technology on Eclipse Luna.  The language meta-model was designed using Ecore and the Eclipse support for OCLinEcore. The three proposed languages allow for describing (i) multi-cloud application requirements; (ii) cloud provider feature models and (iii) cloud provider mappings.

The Eclipse workspace containing the meta-model, grammar and code used to implement the DSLs can be found in the eclipse-workspace subdirectory. The generated plugin files, which can be used to install the modeling tool in Eclipse can be found in lib/plugins/.

To load the workspace or use the editor plugin you need to download Eclipse Luna and install the following software into it:

-       Xtext Complete SDK

-       OCL Classic SDK: Ecore/UML Parsers,Evaluator,Edit

-       OCL Examples and Editors SDK

 

You can either install the plugins by copying the files from lib/plugins/ to the plugins directory of the your Eclipse instalation or execute the editor directly from the code base workspace.

The subdirectory src/main/resources/workspace/ contains an Eclipse workspace with some providers decriptions and application requirements used in our experimentation.

 

Implementation

 

The tool for generating configurations is mostly coded in Groovy with some small portions in Java. It uses the developed Xtext plugin for parsing artifacts, the HermiT OWL Reasoner to reason over ontologies and the Choco Solver to solve feature models as Constraint Satisfaction Problems (CSP).

The tool can be easily built using Gradle, all needed dependencies are either automatically fetched or are included in the bundle. The solve command line script is a wrapper for the fr.inria.spirals.mmcloud.main.Main class that lets you execute the tool to generate multi-cloud configurations from a set of provider descriptions and a requirements file. It includes several parameters for defining the path from where to load files, adding randomly generated providers, logging options, etc. For instance, the following command could be used to generate two configurations and print the output. 

    
./solve -w src/main/resources/workspace/eval/world/ -r src/main/resources/workspace/eval/requirements/Ecommerce.Application.config -n 2 -p

For the experimental evaluation we also created an script that calls solve subsequently to log different parameters from the execution. To reproduce the experiments you can run the script experiment and indicate an output file. Then, you can and use the analyse script to process the results. 

    
$ ./experiment -f output.json

Logging options: All

...
